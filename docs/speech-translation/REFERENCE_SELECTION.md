# Seed-VC参照音声の自動選択メモ

## 現在の軽量版

Seed-VCへ渡す参照音声は、既定では従来どおり先頭から `seed_vc_reference_max_seconds` 秒までをmono WAVへ正規化する。

UIの `参照音声の発話区間を自動選択` をONにした場合は、Seed-VCへ渡す前に軽量な発話区間選択を行う。

- `ffprobe` で参照音声の長さを取得する。
- `ffmpeg silencedetect` で無音区間を検出し、無音ではない区間を発話候補として扱う。
- 候補のうち、上限秒数内で最も長い発話区間を選ぶ。
- 候補が上限秒数より長い場合は、区間の中央寄りから上限秒数ぶんを切り出す。
- 発話候補が取れない、`ffprobe` や `silencedetect` が失敗する、ノイズで無音判定が成立しない場合は、従来どおり先頭から切り出す。

この軽量版は、ノイズを理由に参照音声を空にしない。音声品質を厳密に判定するものではなく、先頭の無音や余計な間を避けるための前処理として扱う。

所要時間はAPIレスポンスの `timings_ms.reference_segment_select` に入れる。UIの所要時間表示で、ONにした場合の追加コストを確認できる。

UIでは、Seed-VC本体を実行する前に参照音声の正規化だけを実行し、正規化前と正規化後をaudio要素で聴き比べられる。翻訳モードでは入力音声、VC比較モードでは参照音声ファイルを対象にする。

## 深掘りする場合の候補

後で品質改善を深める場合は、現在の無音ベース選択を置き換えるか、候補生成だけに使う。

- 参照音声を数秒単位で複数候補に分割する。
- VADで発話区間を取り、短すぎる区間や無音が多い区間だけを弱く減点する。
- speaker embeddingを取り、参照全体のembeddingに近い候補を優先する。
- SNR、クリッピング、音量、ピッチ安定性などを補助スコアにする。
- 最終的にはSeed-VC出力の聴感評価または話者類似度評価と結びつける。

ただし、ノイズ環境の録音を想定するため、ノイズがある候補を一律に除外しない。スコアリングは「よりよい候補を選ぶ」ために使い、候補がなくなる場合は必ず先頭切り出しへ戻す。
