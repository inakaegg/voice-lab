# SpeakLoop LLM比較・採点仕様

更新日: 2026-07-19
状態: 実装前レビュー

## 目的

お手本と復唱の音声認識結果を大規模言語モデル（LLM）へ渡し、フレーズ分割、
認識結果の対応付け、採点を一度に行う。

従来の比較再生処理と文字列類似度による採点は使用しない。

## 処理順

1. お手本音声と復唱音声を既存の音声認識（ASR）で解析し、本文と認識単位ごとの開始・終了時刻を得る。
2. 利用者が選んだLLMへ、目標文、両方の音声認識本文、認識単位ごとの時刻を渡す。
3. LLMが返した値から再生区間と採点表示を作り、処理全体をJSON形式のログへ保存する。

アプリ側では、LLMが返したフレーズ分割と点数を作り直さない。
返された認識単位の位置番号から開始・終了時刻を取り、前後共通の再生余白を加える。

## ユーザー画面

利用者が使う`/speakloop`画面へ次の設定を追加し、復唱録音と一緒にAPIへ送る。

| 設定 | 値 |
| --- | --- |
| 比較モデル | `gpt-5.6-terra`（初期値）、`gpt-5.6-luna`、`gpt-5.4-mini`、`gpt-5.4-nano` |
| 前後共通の再生余白 | 初期値`0.10`秒、`0.00`〜`0.50`秒、`0.05`秒刻み |

## LLM判定結果

LLMは、決められた形式のJSONを返す。

```json
{
  "version": 1,
  "overall_score": 82,
  "feedback": "...",
  "phrases": [
    {
      "target_text": "...",
      "reference_word_start_index": 17,
      "reference_word_end_index": 23,
      "attempt_word_start_index": 17,
      "attempt_word_end_index": 24,
      "phrase_score": 58,
      "feedback": "..."
    }
  ]
}
```

各項目名はAPI内で使う識別子である。

認識単位の範囲は開始位置を含み、終了位置を含まない。該当する復唱箇所がなければ、
復唱側の開始位置と終了位置を`null`にする。全体と各フレーズの点数は、LLMが
`0`〜`100`の整数で決める。「できました」などの表示区分だけは、全体点数から
決められた基準で選ぶ。

LLMに秒数を作らせない。LLMは対応する認識単位の位置番号を選び、アプリが
音声認識結果に含まれる実際の開始・終了時刻を取得する。

## 再生区間

```text
audio_start = max(0, word_start - playback_padding_seconds)
audio_end   = min(audio_duration, word_end + playback_padding_seconds)
```

`word_start`と`word_end`は、LLMが選んだ認識単位の範囲から取得した時刻である。
お手本と復唱の両方に同じ余白を加える。余白によって隣り合うフレーズの再生区間が
重なってもよい。アプリ側で境界を変更しない。

## 比較診断ログ

```text
tmp/practice-llm-alignment/<timestamp>-<id>.json
```

ローカルFastAPIは、復唱1回につき1ファイルを保存する。内容は次のとおり。

- 目標文、モデル、余白
- お手本と復唱の音声認識本文、認識単位ごとの開始・終了時刻
- LLMへ送った内容、LLMから返った未加工の内容、JSONとして読み取った値
- フレーズごとの再生区間、点数、コメント
- API使用量、料金単価、推定料金、処理時間
- 失敗した処理段階と技術的なエラー内容

APIキー、認証情報、音声データそのものは保存しない。Cloudflare公開版は、
この完全なログを保存しない。

## 失敗時

API呼び出しの失敗、JSONの形式不正、認識単位の位置番号の不正が起きた場合は、
旧処理へ自動で切り替えず、比較再生と採点を中止する。

```text
比較結果を作成できませんでした。もう一度お試しください。
```

## 旧処理の扱い

- 新しい比較・採点では、従来の比較再生処理と文字列類似度による採点を呼び出さない。
- 保存済みの旧履歴を表示するために必要な読み取り処理だけを一時的に残す。
- 新しい処理の確認後、旧処理専用の実装、テスト資料、公開文書を削除する。

## 完了条件

- 保存された実データで、お手本の音声認識結果、復唱の音声認識結果、LLMの返却値、
  最終的な再生区間を1つのログで確認できる。
- 4モデルをユーザー画面から選択でき、選択モデルが実際のAPI呼出しへ使われる。
- 前後共通の再生余白がお手本と復唱の全フレーズへ反映される。
- 点数とフレーズ分割をアプリ側で作り直さない。
- LLMによる処理が失敗した場合は旧処理へ切り替えず、指定したエラーを表示する。
- FastAPIとCloudflare Workerが同じ形式の結果を返す。
